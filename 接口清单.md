# 接口清单（与后端实现一致）

> **基准**：`backend/src/main/java/com/dbhelp/controller/*`。  
> 说明：`frontend/src/api.js` 中仍保留对部分路径的封装（如 `/api/tables`、`/api/constraints`、`/api/generate/preview`、`/api/compare/execute`），**当前后端未提供对应 Controller**，调用将返回 404；以下仅列出已实现的接口。

**通用约定**

- 前缀：REST 路径均以 `/api` 开头。
- 内容类型：除特殊说明外，请求/响应为 `application/json`。
- 跨域：后端对 `/api/**` 已配置 CORS（`WebConfig`）。
- 安全：`SecurityConfig` 中对请求均为 `permitAll`，CSRF 关闭（开发态）。

---

## 1. 连接管理 `ConnectionController` — `/api/connections`

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/connections` | 连接列表 |
| `GET` | `/api/connections/{id}` | 按 id 查询单条 |
| `POST` | `/api/connections` | 新建（请求体 `JdbcConnection`，创建校验组） |
| `PUT` | `/api/connections/{id}` | 更新 |
| `DELETE` | `/api/connections/{id}` | 删除；成功返回 `{ "success": true }` 形式（见 `JsonMaps`） |
| `POST` | `/api/connections/test` | 测试连接；请求体为 `Map`，服务端转为 `ConnectionPayload` 后探活；返回 `success` / `message` |
| `GET` | `/api/connections/export` | 导出全部连接配置（JSON） |
| `POST` | `/api/connections/import` | 导入，`multipart/form-data`，字段名 `file`；内容为连接 JSON 数组 |

**说明**：列表与导出接口响应中的密码字段对前端序列化为 **仅写不可读**（`WRITE_ONLY`）；导出文件包含明文密码，请注意保管。

---

## 2. 数据库类型 `DatabaseTypeController` — `/api/database-types`

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/database-types` | 全部数据库类型（含占位符、URL 参数等明细） |
| `GET` | `/api/database-types/{code}` | 按类型编码查询；未知编码返回 400 |

数据来源：应用库表 `database_type` 等，随 `ddl.sql` / `data.sql` 初始化。

---

## 3. 元数据浏览 `DatabaseMetadataController` — `/api/metadata`

面向**目标库**实时查询（非应用库）。请求体使用 `ConnectionPayload`、`TablesMetadataRequest`、`ColumnsMetadataRequest` 等 DTO。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/metadata/catalogs` | 列出 catalog / database（因方言而异） |
| `POST` | `/api/metadata/tables` | 列出指定库（可多库）下的表 |
| `POST` | `/api/metadata/columns` | 列出指定库、表的列 |

方言路由见 `MetadataDialectRegistry`：`MYSQL` / `MARIADB` / `GOLDENDB`、`POSTGRESQL`、`SQLSERVER`，其余走 `DefaultMetadataDialect`。

---

## 4. 约束解析 `ConstraintController` — `/api/constraint`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/constraint/soft/parse`（含尾部 `/` 别名） | **软约束**：解析 SQL，使用 `com.jsjjlt.sqlparser`；请求体 `SqlParseRequest`（含 `sql`、`prefix`、`singleQualifierRole`、`strict`）；返回 `constraints`、`errors`、`parseOk` |
| `POST` | `/api/constraint/hard/parse`（含尾部 `/` 别名） | **刚性约束**：请求体 `ColumnsMetadataRequest`；从库拉列并与方言解析器配合；**首期刚性推导针对 MySQL `information_schema`**（见 `RigidConstraintService`） |

---

## 5. 造数任务 `GenerateTaskController` — `/api/generate/tasks`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/generate/tasks` | 创建任务；请求体 `GenerateTaskCreateRequest`（名称、`connectionMode`、`connectionId` 或 `inlineConnection`、`targets`、软硬约束列表、`options` 等） |
| `GET` | `/api/generate/tasks` | 分页列表；查询参数：`name`、`description`、`status`、`page`（默认 1）、`pageSize`（默认 10）；响应 `GenerateTaskPageDto` |
| `GET` | `/api/generate/tasks/{id}` | 任务详情 `GenerateTaskDetailDto` |
| `DELETE` | `/api/generate/tasks/{id}` | 删除任务 |
| `GET` | `/api/generate/tasks/{id}/metrics` | 指标 `GenerateTaskMetricsDto` |
| `POST` | `/api/generate/tasks/{id}/start` | 启动执行 |
| `POST` | `/api/generate/tasks/{id}/retry` | 重试；可选 JSON `{ "resumeFromCheckpoint": true/false }` |
| `POST` | `/api/generate/tasks/{id}/cancel` | 取消 |

造数流水线：`GenerateTaskService` → `DataFakerGeneratePipeline`；`options.transport` 为 `rabbitmq` 时需配置 `dbhelp.generate.rabbit.enabled=true` 且 RabbitMQ 可用；`direct` 为进程内写入。详见 `application.yml` 与 `README.md`。

---

## 6. 其他

- **当前仓库无** `/api/tables`、`/api/constraints` CRUD、`/api/generate/preview`、`/api/compare/*`、`/api/sql/*`、`/api/projects/*` 等 Controller。
- **Actuator**：若未在配置中关闭，Spring Boot 可能暴露 `/actuator/*`，不属于业务 REST 契约，集成测试时请按需忽略或关闭。
