# dbhelp 需求文档（基于当前实现修订）

## 一、项目概述

### 1.1 项目名称与定位

**dbhelp / database-test-tool**：面向数据库开发与测试的 Web 工具，提供 JDBC 连接管理、目标库元数据浏览、SQL 约束解析与造数任务编排等能力。采用 **前后端分离**：Vue 3 + Vite 前端，Spring Boot 2.7 + MyBatis-Plus 后端。

### 1.2 目标用户

- 数据库开发与测试人员  
- 需要在测试库中批量造数的工程师  

### 1.3 文档范围说明

本文档区分 **已实现能力** 与 **规划或未实现能力**。规划项来源于早期产品设想或前端残留接口封装，**不代表当前里程碑承诺**。

## 二、已实现需求（当前代码）

### 2.1 数据库连接管理

| 需求 | 实现说明 |
|------|----------|
| 维护多条 JDBC 连接配置 | 已实现：CRUD，持久化至应用库表 `jdbc_connection` |
| 数据库类型驱动选择 | 已实现：`GET /api/database-types`，类型、占位符、URL 模板来自 `database_type` 等表 |
| 连接测试 | 已实现：`POST /api/connections/test`，按请求体组装连接并探活 |
| 导入/导出连接 | 已实现：JSON 导出；文件导入 multipart |
| 密码加密存储 | **当前未实现**：库中密码为明文保存；响应序列化对密码字段 **不写回前端**（降低泄露面）。生产环境建议叠加加密或外部密钥库 |

### 2.2 目标库元数据

| 需求 | 实现说明 |
|------|----------|
| 浏览 catalog / database | 已实现：`POST /api/metadata/catalogs` |
| 浏览表列表 | 已实现：`POST /api/metadata/tables` |
| 浏览列定义 | 已实现：`POST /api/metadata/columns` |
| 方言差异 | 已实现：`MetadataDialectRegistry` 对 MySQL 系、PostgreSQL、SQL Server 等分支；其它 JDBC 库走默认方言（能力可能受限） |

### 2.3 约束解析（服务于造数）

| 需求 | 实现说明 |
|------|----------|
| 从 SQL 文本解析软约束（WHERE 等） | 已实现：`POST /api/constraint/soft/parse`，底层 `com.jsjjlt.sqlparser` |
| 从库表推导刚性约束（范围/等值等） | 已实现：`POST /api/constraint/hard/parse`；**首期刚性推导以 MySQL information_schema 路径为主**（见 `MysqlRigidConstraintResolver`） |

### 2.4 造数任务

| 需求 | 实现说明 |
|------|----------|
| 创建造数任务（目标表、行数、并发、批次、sink 等） | 已实现：`POST /api/generate/tasks`，配置见 `GenerateTaskCreateRequest` / `GenerateOptionsDto` |
| 任务列表与详情、进度与指标 | 已实现：分页列表、详情、`metrics` |
| 启动 / 重试 / 取消 | 已实现：对应子路径 |
| 写入方式 | JDBC 批量插入或 CSV 追加（`sinkType`）；传输模式 `direct` 或 `rabbitmq`（启用 Rabbit 时需中间件与配置） |

### 2.5 前端界面（当前路由）

| 需求 | 实现说明 |
|------|----------|
| 主导航 | **连接管理**、**造数任务**（列表与新建在同一模块内 Tab 切换），见 `App.vue`、`router.js` |
| 其它 `.vue` 页面 | 仓库中仍存在 `TablePage`、`ConstraintPage` 等文件；**若未挂载路由则对用户不可达**，是否接线以仓库为准 |

## 四、非功能需求（与实现对齐）

| 类别 | 说明 |
|------|------|
| 性能 | 大批量造数依赖批次、`transport` 与可选 MQ；具体 SLA 需结合硬件与目标库 |
| 兼容性 | 后端要求 JDK 8+；前端构建要求 Node 16+；浏览器访问现代 Chromium / Firefox / Safari |
| 安全 | 当前接口默认匿名可访问；数据库凭证不落前端列表响应，但存储与传输策略需在部署层加固 |
| 可扩展性 | 元数据方言可新增实现类并注册；造数管线与 Sink 可扩展 |
