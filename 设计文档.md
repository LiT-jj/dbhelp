## 一、技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3、Vite 5、Vue Router、Pinia、Element Plus、Axios | 单页应用，开发端口默认 5173 |
| 后端 | Spring Boot **2.7.18**、Java **8** | 主类 `DatabaseTestToolApplication` |
| 持久化 | MyBatis-Plus 3.5.x | 应用业务表存 MySQL（由 `application.yml` 与 `sql/ddl.sql` 管理） |
| 安全 | Spring Security | 全 URL 放行、CSRF 关闭、HTTP Basic 配置存在但业务 API 不强制认证（`SecurityConfig`） |
| 解析 | `com.jsjjlt:sqlparser:1.0` | 软约束 SQL 解析（非 JSqlParser） |
| 造数 | DataFaker 1.9.0 | 假数据生成 |
| 消息 | Spring AMQP（可选） | 主类 **排除** `RabbitAutoConfiguration`；仅在 `dbhelp.generate.rabbit.enabled=true` 时装配自定义 Rabbit 配置 |
| 驱动 | MySQL、PostgreSQL、Oracle（ojdbc8）、H2（runtime）等 | 以 `pom.xml` 为准 |

## 二、总体架构

采用 **前后端分离**：浏览器访问前端静态资源或 Vite 开发服务器；通过 HTTP JSON 调用后端 REST。

```mermaid
flowchart LR
  subgraph browser [浏览器]
    UI[Vue SPA]
  end
  subgraph backend [Spring Boot]
    C[Controller]
    S[Service]
    G[Generate Pipeline]
    M[MyBatis Mapper]
  end
  subgraph appdb [(应用库 MySQL)]
    T1[(jdbc_connection 等)]
    T2[(generate_task)]
  end
  subgraph target [(目标库)]
    TJ[(用户业务库)]
  end
  subgraph mq [可选 RabbitMQ]
    Q[Queue]
  end
  UI -->|REST /api| C
  C --> S
  S --> M
  M --> T1
  S --> G
  G -->|JDBC 写入| TJ
  G -.->|transport=rabbitmq| Q
  Q -.->|消费写入| TJ
```

## 三、后端分层与包结构（概要）

| 包/层 | 职责 |
|------|------|
| `controller` | REST 入口：`Connection`、`DatabaseType`、`DatabaseMetadata`、`Constraint`、`GenerateTask` |
| `service` | 业务编排：`JdbcConnectionService`、`DatabaseTypeService`、`DatabaseMetadataService`、`RigidConstraintService`、`GenerateTaskService` 等 |
| `mapper` | MyBatis：应用库表 CRUD |
| `entity` / `dto` | 实体与传输对象 |
| `service.metadata.dialect` | 按数据库品种拆分元数据查询 |
| `generate.*` | 造数流水线、Sink、Rabbit 批次、Faker、类型与约束归一 |
| `config` | Security、Web/CORS、异步线程池等 |
| `exception` | 全局异常处理 |

## 四、核心模块设计

### 4.1 连接管理

- **模型**：`JdbcConnection` 映射表 `jdbc_connection`（名称、dbType、主机端口、库名、用户名密码、驱动与 URL 模板等）。
- **校验**：`ValidationGroups.Create` / `Update` 分组校验。
- **密码**：请求可写入；JSON **响应**中密码字段 `WRITE_ONLY`，避免列表泄露。
- **测试**：`JdbcConnectionTester` 将 Map / `ConnectionPayload` 转为 JDBC URL 并探测连接。

### 4.2 数据库类型与 URL 拼装

- **模型**：`database_type`、`database_type_placeholder`、`jdbc_url_parameter`。
- **服务**：`DatabaseTypeService` 提供类型列表与按 code 查询。
- **连接解析**：`JdbcUrlBuilder`、`ResolvedConnection` 根据模板与占位符生成最终 JDBC URL。

### 4.3 目标库元数据

- **入口**：`/api/metadata/catalogs|tables|columns`，请求体携带目标连接与 catalog/schema/table 上下文。
- **服务**：`DatabaseMetadataService` 按 `dbType` 选择 `MetadataDialect`：
  - `MYSQL` / `MARIADB` / `GOLDENDB` → `MysqlFamilyMetadataDialect`
  - `POSTGRESQL` → `PostgresqlMetadataDialect`
  - `SQLSERVER` → `SqlServerMetadataDialect`
  - 其它 → `DefaultMetadataDialect`（基于 JDBC `DatabaseMetaData` 的通用逻辑，能力依驱动而异）

### 4.4 约束解析

- **软约束**：`ConstraintController` → `JsqlParser` → `SqlParseResultMapper`，输出表级约束 DTO、错误列表、`parseOk`。
- **刚性约束**：`RigidConstraintService` 接收列元数据请求，针对 MySQL 系走 `MysqlRigidConstraintResolver`（`information_schema`）；返回列约束条目。

### 4.5 造数任务

- **持久化**：表 `generate_task`（状态、进度、`config_json`、`checkpoint_json`、错误与告警 JSON 等）。
- **生命周期**：创建 → 启动 → 运行中 / 成功 / 失败 / 取消；支持重试（是否从 checkpoint 恢复由请求体控制）。
- **执行**：`GenerateTaskAsyncExecutor` 异步执行；核心逻辑 `DataFakerGeneratePipeline`：
  - 解析任务配置中的目标表与软硬约束；
  - `FakedValueProducer` 结合类型与 `ColumnConstraintHints` 产出行；
  - **Sink**：`JdbcBatchInsertSink` 或 `CsvAppendSink`；
  - **transport**：`direct` 在同进程写入；`rabbitmq` 将批次投递至队列，由 `GenBatchListener` 消费写入（需启用 Rabbit 相关 Bean）。
- **指标**：`GenerateTaskMetricsRegistry` 等支撑 `/metrics` 查询。

### 4.6 跨域与安全

- **CORS**：`WebConfig` 对 `/api/**` 放开浏览器的跨域调用（开发模式下前端独立端口）。
- **安全**：当前为开发友好策略；生产部署应在前缘网关或 Spring Security 上收紧认证与 HTTPS。

## 六、配置要点（后端）

| 配置项 | 含义 |
|--------|------|
| `spring.datasource.*` | 应用库（MySQL），存放连接配置与任务 |
| `spring.sql.init.*` | 启动执行 `classpath:sql/ddl.sql` 与 `data.sql` |
| `dbhelp.generate.transport` | 默认造数传输策略示例（任务内 `options.transport` 可覆盖） |
| `dbhelp.generate.rabbit.*` | RabbitMQ 开关、连接、队列名等（`enabled=false` 时不装配监听） |

详见项目根目录 `README.md`。

## 八、已知限制与设计债务（文档化）

| 项目 | 说明 |
|------|------|
| 密码存储 | 应用库中明文；依赖操作系统与数据库权限隔离 |
| 前端 API 封装 | `api.js` 存在未对接后端的占位方法 |
| 刚性约束品种 | 首期 MySQL 刚性推导完整度高于其它方言 |
| 比对 / SQL 控制台 | 无后端模块 |

以上为当前实现边界，迭代时优先更新本文档与 `接口清单.md`。
